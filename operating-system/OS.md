# 운영체제

## 운영체제란
  - 컴퓨터 하드웨어 위에 설치되어 사용자 및 다른 소프트웨어와 하드웨어를 연결하는 소프트웨어 계층
  - 하드웨어를 직접 관리
  - 컴퓨터 시스템을 편리하게 활욜할수 있게 제공
  - 컴퓨터 시스템의 자원을 효율적으로 관리<br/>
  ( CPU스케줄링 , MEMORY 관리 , I/O장치 디스크 스케줄링 , CPU와 I/O 장치간의 속도차 극복 인터럽트, 캐싱)


## CPU 스케줄링
  1. FCFS(First-Come First-Serve) <br/>
    - 도착 순서대로 처리 
  2. SJF(Shortest-Job-First) <br/>
    - 금번 CPU 사용시간이 가장 짧은 순서대로<br/>
    - minimum average waiting time 보장<br/>
    - starvation(기아현상) 발생 가능
  3. RR(Round-Robin) <br/>
    - 각 프로세스는 동일 크기의 CPU 할당 시간을 가짐
    - 시간이 끝나면 인터럽트가 발생해 다시 제일 뒤로 보냄
    - n개의 프로세스가 있는 경우 (n-1)*할당시간 이상 기다리지 않음 
    - 대기시간이 프로세스의 CPU 사용시간에 비례함

## 메모리 관리  
  - 각 프로세스는 각자의 가상메모리가 있음 그 가상메모리에서 필요한 부분만 실제 메모리에 올려서 사용
  1. LRU <br/>
    - 가장 오래전에 참조 페이지 삭제
  2. LFU <br/>
    - 참조횟수가 가장 적은 페이지 삭제

## 디스크 스케줄링
  - Seek Time을 줄이는 것이 목표
  - 디스크 접근시간(Access time) 구성
    - 탐색시간(Seek Time) : 헤드를 트랙으로 옮기는 데 걸리는 시간
    - 회전지연(Rotational latency) : 헤드가 원하는 섹터에 도달하는 데 걸리는 시간
    - 전송시간(Transfer Time) : 실제 데이터의 전송 시간
